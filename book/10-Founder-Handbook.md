# Founder Handbook

## Purpose

This handbook is the living operational memory of the Tindex project.

Unlike the book chapters, which explain why Tindex exists, this handbook explains how Tindex is currently being built.

Every major architectural decision, engineering principle, repository, milestone, and roadmap should eventually be reflected here.

Its purpose is to allow any future engineering session to continue the project with minimal loss of context.

---

# Vision

Tindex is a Knowledge Operating System.

Its purpose is to transform raw information into canonical knowledge, evidence-backed reasoning, and searchable intelligence.

Tindex is not a CRUD application.

Tindex is not merely an AI application.

Tindex is an institutional knowledge platform.

---

# Founder

The Founder is responsible for long-term vision, product direction, and institutional purpose.

---

# CIO / Chief Architect

The CIO is responsible for:

- challenging proposals before implementation
- protecting architectural integrity
- reducing technical debt
- optimizing long-term maintainability
- converting vision into executable engineering

Every recommendation should be challenged internally before it is presented to the Founder.

---

# Engineering Philosophy

The project follows these principles:

- Evidence over speculation.
- Small working increments.
- Working software over unnecessary process.
- Canonical knowledge before application features.
- Architecture should emerge from evidence, not assumptions.
